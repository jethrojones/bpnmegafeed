# Mega RSS Feed Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a configurable podcast RSS aggregator that preserves original episode enclosure URLs.

**Architecture:** A small Node.js CLI reads feed URLs and network config, fetches source RSS feeds, parses and normalizes episodes, merges and deduplicates them, then writes static RSS XML. A lightweight HTTP server previews the generated feed locally.

**Tech Stack:** Node.js, `fast-xml-parser`, built-in `node:test`, built-in `fetch`, built-in HTTP server.

---

## File Structure

- `package.json`: scripts and dependency metadata.
- `feeds.json`: source podcast feed URL list.
- `mega-feed.config.json`: network RSS metadata and build settings.
- `src/feed-sources.js`: load JSON config files and validate values.
- `src/parse-source-feed.js`: parse source RSS XML into normalized feed/item objects.
- `src/merge-feeds.js`: merge, deduplicate, sort, and limit episodes.
- `src/rss-xml.js`: generate final RSS XML with original enclosure URLs.
- `src/build-feed.js`: build command entrypoint.
- `src/server.js`: local preview server.
- `test/parse-source-feed.test.js`: parser behavior tests.
- `test/merge-feeds.test.js`: merge behavior tests.
- `test/rss-xml.test.js`: RSS generation behavior tests.
- `README.md`: usage, adding shows, hosting guidance.
